# Manor — Design Context

*For Impeccable and its action skills. Loaded automatically by `impeccable craft`, `shape`, `critique`, etc.*

---

## Users

**Manor is one person's tool.** Hana — writer, novelist, NYT-featured. Uses it on her macOS desktop, privately, every day. Nobody else logs in. No accounts, no onboarding, no marketing surfaces.

**Context of use:**
- Morning review at a desk over coffee, evening glance before bed.
- While writing (so: must never interrupt flow, never beg for attention).
- Side-by-side with Scrivener, Pages, Ulysses — Manor lives in the same workspace as those, native-feeling.

**Jobs to be done:**
- Hold the day in one place (tasks, calendar, rhythm, chores, ledger).
- Reflect on the month (MonthReviewPanel, AI narrative).
- Track household state without turning into a spreadsheet.
- Keep a calm, private relationship with time and money.

---

## Brand Personality

**Three words:** *flat*, *minimal*, *quiet*.

Manor is a well-kept notebook — document-flat, not dashboard-shaped. It reads left-to-right like prose, with small inline icons that provide structure without chrome. Text does most of the work; surfaces get out of the way.

**Emotional goal:** calm, legible, at-rest. Opening Manor should feel like opening a clean page — not an app loaded with badges and KPIs.

**Voice in the UI:** plain, short, specific. "Events" not "SCHEDULED APPOINTMENTS". "£760 remaining" not "Budget Status: 62% utilised". No chirpy tone, no shouty caps, no "Great job!" toasts.

## Aesthetic Direction

**Flat-Notion with icons.** Apple-HIG native DNA (SF Pro, system-ish semantics, macOS-native feel) with Vercel-flat execution (hairlines over cards, small radii, monochrome). Layered with Lucide outline icons for section identity, buttons, and navigation — the Notion reference point.

**Light:**
| Token | Value | Use |
|---|---|---|
| `--paper` | `#fcfcfc` | Page background — warm-white, not pure |
| `--surface` | `#ffffff` | Rare card bg (SummaryCard pullquote only) |
| `--ink` | `#1f1f1f` | Primary text, primary button bg |
| `--ink-soft` | `#6b6b6b` | Labels, metadata, icons |
| `--ink-faint` | `#8a8a8a` | Disabled, line-through text |
| `--hairline` | `#efefef` | Row dividers |
| `--hairline-strong` | `#e0e0e0` | Button borders, drawer edges |

**Dark:**
| Token | Value | Use |
|---|---|---|
| `--paper` | `#191919` | Page background — warm-near-black |
| `--surface` | `#1c1c1e` | Rare card bg |
| `--ink` | `#e8e8e8` | Primary text |
| `--ink-soft` | `#9a9a9a` | Labels, metadata, icons |
| `--ink-faint` | `#636366` | Disabled |
| `--hairline` | `#2a2a2a` | Row dividers |
| `--hairline-strong` | `#3a3a3a` | Button borders |

**Action colors** invert per theme — primary button is ink-on-paper in light, paper-on-ink in dark. No blue, no green, no rust. Fully monochrome. A single muted accent may be introduced later based on use, but is not in the current spec.

### Typography

- **Body:** macOS system stack (`-apple-system, BlinkMacSystemFont, "SF Pro Text"`). No web fonts.
- **Numbers:** `ui-monospace, "SF Mono"` — everywhere numeric (times, currency, counts, durations, IDs). `font-variant-numeric: tabular-nums` applied globally via `[data-num]` / `.num` utility.
- **No web fonts.** `@fontsource/*` imports are removed. The app loads instantly, respects user Display → Text Size, and feels native next to every other macOS app.

Type scale (fixed rem — fluid type is for marketing, not app UI):

```
--text-xs:  0.75rem   (12px) — section labels, metadata, pill captions
--text-sm:  0.8125rem (13px) — list row body
--text-md:  0.875rem  (14px) — primary body, default
--text-lg:  1rem      (16px) — emphasis
--text-xl:  1.375rem  (22px) — page titles (600 weight, -0.015em tracking)
--text-2xl: 1.75rem   (28px) — rare hero moments
```

Weight usage: 400 default, 500 for section labels and buttons, 600 for page titles and strong numbers. No 700/800. Letter-spacing only on `--text-xl` and up (`-0.015em`).

### Icon language — Lucide

- **Library:** `lucide-react` (single library, no mixing).
- **Page identifiers:** 22px, `strokeWidth={1.8}`, text-colored, to the left of every view title.
- **Section labels:** 14px, `strokeWidth={1.8}`, tinted `--ink-soft`, beside every section introducer.
- **Buttons:** 12px, `strokeWidth={2.2}`, inherits button color.
- **List affordances** (checkboxes, chevrons): 14px, `strokeWidth={1.8}`.

Canonical mappings are in the implementation spec. Icons always use `currentColor` fill — they tint with their parent text automatically. No colored icons. No emoji in chrome (emoji stays inside user content: notes, reactions, user-set category markers).

### Spatial

- 4pt scale: 4, 8, 12, 16, 22, 28, 42.
- **No cards.** Sections separated by `[icon] [label]` introducers, whitespace, and 1px `--hairline` dividers between rows.
- **Generous vertical rhythm.** 22–28px between major sections.
- Column width capped ~70ch for text-heavy surfaces (MonthReviewPanel narrative).
- Radii: 4px tight, 5px buttons/inputs, 6px cards. No pills on buttons.

### Motion

- `ease-out` cubic-bezier `(0.2, 0.8, 0.2, 1)`.
- Durations: fast 120ms, medium 200ms. No 300ms+ anywhere.
- No bounce, no elastic, no overshoot.
- `transform` and `opacity` only — never `width`, `height`, `padding`, `margin`.
- `prefers-reduced-motion: reduce` collapses everything to 0.01ms. Non-negotiable.

### Surface philosophy

Manor is **document-flat**:
- List views have no cards. Just sections introduced by icon + label, lists flowing underneath, hairline dividers between rows.
- The only card-like surface is `SummaryCard` in Ledger — a deliberate dark-on-paper pullquote for emphasis. Treated as a solid `--action-bg` block.
- Drawers (Settings modal, ConnectBankDrawer, ConversationDrawer) use `--surface` and hairline borders; no backdrop-blur.

## Design Principles

1. **Text does the work.** Icons are small punctuation, not focal elements. Color is rare. Hierarchy comes from weight, size, and space.
2. **Document-flat over card-based.** If a surface starts needing a card to feel "grouped," it probably needs better section introducers instead. Cards are a last resort.
3. **Monochrome until proven otherwise.** No colored accent ships in Phase 6. If lack of color proves too stark after a week of use, introduce exactly one muted accent — not a palette.
4. **Native, not distinctive.** Manor should feel like a first-party macOS app, not a branded product. No custom fonts, no custom chrome. Let the system do its job.
5. **Numbers deserve a typographic voice.** All numeric content uses SF Mono with `tabular-nums`. Times, currency, counts — consistent across every view.
6. **One icon library.** Lucide, always. No Heroicons, no Feather, no emoji-in-chrome. Consistent strokeWidth 1.8. `currentColor` fill.
7. **Auto-respect system preference.** Light / dark switches automatically via `prefers-color-scheme`. No manual theme toggle unless a future need appears.
8. **Anti-corporate, anti-chirpy.** No "Great job!" toasts, no confetti, no badges, no gradient text, no uppercase section labels. Manor doesn't high-five.

## Anti-References

What Manor must *never* look like:

- **SaaS dashboards** — gradient KPI tiles, identical cards in grids, charts everywhere.
- **AI tech-bro** — dark-mode-with-cyan-glow, purple-to-blue gradients, "AI-powered" badges, sparkle icons.
- **Corpo productivity** — Microsoft 365 chrome, enterprise-safe grays, Inter everywhere.
- **Consumer iOS cloneware** — rounded cards nested in rounded cards, pastel gradients, bounce animations, iMessage-style chat bubbles.

Technical anti-patterns (enforced by `audit`):

- Gradient text (`background-clip: text`).
- Coloured side-stripe borders (`border-left: Npx solid …` where N > 1).
- Glassmorphism as decoration (`backdrop-filter: blur(…)` outside genuine modal scrims).
- Reflex fonts — Inter, Roboto, Fraunces, Playfair, DM Sans, IBM Plex, Space Grotesk, Outfit, Nunito, Geist (banned as root font), Cormorant, Syne, Crimson.
- Pure `#000` / `#fff` text or background.
- Cool-neutral grays without a hue token.
- Bounce / elastic easing.
- Pills (`border-radius: 9999px`) on regular buttons.
- Icons in rounded-square background chrome above headings.
- Emoji inside app chrome (nav icons, section markers, status pills).
- Modals when a side panel or inline reveal would work better.

## Accessibility

- macOS desktop, single user.
- **Reduced motion:** respected globally. Non-negotiable.
- **Contrast:** WCAG AA body (4.5:1), AAA target for primary text (7:1+). `#1f1f1f` on `#fcfcfc` is ~15:1; `#e8e8e8` on `#191919` is ~13:1. Generous margin.
- **Dynamic Type:** respect macOS Display → Text Size. No viewport-locked `px` for body text.
- **Color-independence:** never rely on color alone. Every ivy/rust-style distinction (where accent is later introduced) must be paired with an icon, label, or shape cue.
- **Focus:** `:focus-visible` outline everywhere interactive. `outline: none` without a replacement is banned.
- **Keyboard:** every `role="button"` element has an `onKeyDown` handler for Enter/Space.

## Current Migration (as of 2026-04-17)

Phase 6 — Flat Design System migration — replaces the entire previous token + typography system:

- **Nunito** (root font) retired. Body is SF Pro via system stack.
- **iMessage tokens** (`--imessage-blue`, `--imessage-green`, `--imessage-red`) retired. Primary action becomes monochrome ink-on-paper / paper-on-ink.
- **Phase-5 dark-blue gradient** (`#1a1a2e → #16213e`) on SummaryCard + ConnectBankDrawer + BankAccountRow retired. Replaced with `--action-bg` solid.
- **Bordered rounded cards** on Today, Ledger, Chores, TimeBlocks, Settings list views retired. Replaced with section-introducer pattern (icon + label + hairline-separated rows).
- **iMessage chat bubbles** (BubbleLayer) deleted. ConversationDrawer restyled: hairline-separated paragraphs with role labels.
- **Lucide icons** added at every page title, section label, button, and list affordance. Single library.
